# 벨로컴퍼니 홈페이지 — 수정·검증 후 실서비스 반영 가이드 (초보자용)

개발을 모르는 분도 따라 할 수 있도록, **한 폴더**에서 **먼저 bellodev에 올려서 검증**하고, **문제 없을 때만 bello(실서비스)에 반영**하는 방법을 단계별로 안내합니다.

---

## 1. 이렇게 하는 이유 (한 줄 요약)

- **bello** = 지금 실사용자들이 보는 **bellocompany.co.kr** (건드리면 바로 반영됨)
- **bellodev** = 수정·개선을 **먼저 올려서 확인**하는 **테스트용** 저장소
- 같은 컴퓨터의 **‘마케팅사 홈페이지’ 폴더 하나**만 사용하고,  
  **“먼저 bellodev에 올리기 → 확인 → 괜찮으면 bello에 올리기”** 순서로 하면 실서비스는 안전하게 유지됩니다.

---

## 2. 항상 작업하는 폴더 (하나만 쓰세요)

- **항상 이 폴더만** 사용하세요:  
  `바탕화면\마케팅사 홈페이지`  
  (또는 Cursor에서 열어 둔 프로젝트 폴더)
- **‘복사본’ 폴더는 사용하지 않아도 됩니다.**  
  같은 폴더에서 “bellodev에 올리기”와 “bello에 올리기”를 구분해서 쓰면 됩니다.

---

## 3. 전체 흐름 (3단계)

```
① 수정하기 (파일 저장)
     ↓
② bellodev에 올리기 (테스트용) → 웹사이트/Netlify에서 확인
     ↓
③ 문제 없으면 bello에 올리기 (실서비스 반영)
```

아래는 이 3단계를 **매우 구체적으로** 적은 것입니다.

---

## 4. ① 수정하기 (파일 저장)

1. Cursor(또는 VS Code)에서 **‘마케팅사 홈페이지’** 프로젝트를 연다.
2. 왼쪽 **파일 목록**에서 수정할 파일을 클릭해 연다.
3. 내용을 수정한 뒤 **저장**한다.  
   - 단축키: `Ctrl + S` (Windows)
4. 수정이 끝날 때마다 저장하는 습관을 들이면, 나중에 “어디까지 반영했지?” 헷갈리지 않습니다.

---

## 5. ② bellodev에 올리기 (테스트용으로 먼저 반영)

수정한 내용을 **실서비스(bello)가 아닌, 테스트용(bellodev)** 저장소에만 올리는 단계입니다.

### 5-1. 터미널 여는 방법

- **Cursor / VS Code 사용 시**
  1. 상단 메뉴에서 **터미널(Terminal)** 클릭
  2. **새 터미널(New Terminal)** 선택  
  → 화면 아래쪽에 검은 창(또는 흰 창)이 뜹니다.
- **또는** 단축키: **Ctrl + `** (백틱, 숫자 1 왼쪽 키)

### 5-2. 프로젝트 폴더로 이동

터미널에 **아래 한 줄**을 **그대로** 입력한 뒤 **Enter**를 누릅니다.

```bash
cd "C:\Users\yhs17\OneDrive\Desktop\마케팅사 홈페이지"
```

- `cd` = “이 폴더로 이동”  
- 경로가 다르다면, 본인 컴퓨터의 **‘마케팅사 홈페이지’ 폴더 경로**로 바꿔서 입력하면 됩니다.

### 5-3. 수정한 내용을 “커밋”으로 남기기

**한 줄씩** 입력하고, 각 줄 입력 후 **Enter**를 누릅니다.

**1) 수정된 파일을 스테이징**

```bash
git add .
```

- 의미: “지금 폴더에서 수정한 파일들을, 다음에 올릴 목록에 넣는다.”

**2) 커밋 만들기**

```bash
git commit -m "수정 내용 요약"
```

- `"수정 내용 요약"` 부분은 **본인이 이해할 수 있는 짧은 문장**으로 바꿔서 입력합니다.  
  예: `"상단 문구 변경"`, `"연락처 수정"`, `"2월 업데이트"`

### 5-4. bellodev(테스트용)에 올리기

```bash
git push bellodev main
```

- 의미: “지금까지 커밋한 내용을 **bellodev**라는 이름의 원격 저장소(GitHub)의 **main** 브랜치로 보낸다.”
- **bello(실서비스)에는 전혀 영향이 없습니다.**  
  실사용자가 보는 bellocompany.co.kr는 그대로입니다.

### 5-5. 올라갔는지 확인하는 방법

1. 브라우저에서 **https://github.com/yhs1780-arch/bellodev** 로 접속
2. **가장 위에 있는 파일 목록·커밋 메시지**가 방금 한 수정과 맞는지 확인

**Netlify를 bellodev와 연결해 두었다면**  
- bellodev에 푸시한 뒤 몇 분 지나면, Netlify에 설정한 “bellodev용 사이트” 주소에서 수정된 화면을 볼 수 있습니다.  
- 그 주소에서 꼼꼼히 확인(QA)하면 됩니다.

---

## 6. ③ 문제 없을 때만 bello(실서비스)에 올리기

bellodev에서 확인했을 때 **이상 없고, 이제 실사이트에도 반영해도 되겠다**고 판단했을 때만 진행합니다.

### 6-1. 터미널에서 같은 프로젝트 폴더인지 확인

아래처럼 **이미 ②에서 사용한 터미널**이 그대로라면, 같은 폴더에 있을 가능성이 높습니다.  
다른 폴더에 있다면, 다시 한 번:

```bash
cd "C:\Users\yhs17\OneDrive\Desktop\마케팅사 홈페이지"
```

로 이동합니다.

### 6-2. bello(실서비스)로 푸시

**한 줄만** 입력합니다.

```bash
git push origin main
```

- 의미: “지금 **main** 브랜치에 있는 내용을 **origin**(= bello, 실서비스용 GitHub 저장소)으로 보낸다.”
- 이렇게 하면 **bellocompany.co.kr**에 연결된 Netlify가 새로 배포를 하면서, 실사이트에 수정 내용이 반영됩니다.

### 6-3. 실서비스 반영 확인

- 브라우저에서 **https://bellocompany.co.kr** 접속
- 수정한 부분이 반영되었는지 확인

---

## 7. dev 적용이 마음에 안 들 때 — 기존 main으로 되돌리기

수정·검증 중에 **“이대로는 안 되겠다, 예전 실서비스 상태로 돌아가고 싶다”** 할 때 사용합니다.

- **bello(origin)에는 아직 푸시하지 않았다**고 가정합니다.  
  → 실사이트(bellocompany.co.kr)는 그대로이고, **내 컴퓨터 폴더만** 예전 main 상태로 맞추면 됩니다.

### 7-1. 터미널 열기

- Cursor에서 **Ctrl + `** 로 터미널을 엽니다.

### 7-2. 프로젝트 폴더로 이동

```bash
cd "C:\Users\yhs17\OneDrive\Desktop\마케팅사 홈페이지"
```

### 7-3. GitHub의 실서비스(bello) 상태 가져오기

```bash
git fetch origin
```

- 의미: “origin(bello) 저장소의 **최신 상태**를 가져온다.” (아직 폴더 내용은 안 바뀜)

### 7-4. 로컬을 실서비스(main) 상태로 완전히 맞추기

```bash
git reset --hard origin/main
```

- 의미: “지금 폴더를 **origin의 main**과 똑같이 만든다. 로컬에서 한 수정·커밋은 **전부 사라진다**.”
- **주의:** 실행 후에는 **되돌릴 수 없습니다.** 정말 dev 적용을 버리고 돌아갈 때만 사용하세요.

### 7-5. 확인

- Cursor에서 파일을 열어 보면, **실서비스(bellocompany.co.kr)와 같은 내용**으로 되어 있어야 합니다.
- 다음에 다시 수정할 때는 이 상태에서 새로 작업하면 됩니다.

**요약:**  
dev 쪽 수정이 마음에 안 들면 → `git fetch origin` → `git reset --hard origin/main` 하면 **로컬만** 기존 main(실서비스) 상태로 돌아갑니다. bello에는 푸시하지 않았으므로 **실사이트는 원래부터 그대로**입니다.

---

## 8. 자주 하는 실수와 주의사항

| 하지 말아야 할 것 | 해야 할 것 |
|------------------|------------|
| 수정 직후 `git push origin main`만 하기 | 먼저 `git push bellodev main` → 확인 → 괜찮을 때만 `git push origin main` |
| ‘복사본’ 폴더에서 수정한 뒤 본 폴더에 수동으로 복사하기 | **항상 ‘마케팅사 홈페이지’ 한 폴더**에서만 수정하고, bellodev / bello만 구분해서 푸시 |
| `git push`만 입력하기 (대상 지정 없이) | **테스트용:** `git push bellodev main` / **실서비스:** `git push origin main` |

## 9. 명령어 한눈에 보기

```bash
cd "C:\Users\yhs17\OneDrive\Desktop\마케팅사 홈페이지"
git add .
git commit -m "여기에 수정 내용 요약"
git push bellodev main
```

**실서비스(bello)에 반영할 때 (검증 후 한 번만)**

```bash
cd "C:\Users\yhs17\OneDrive\Desktop\마케팅사 홈페이지"
git push origin main
```

**dev 적용이 마음에 안 들어서 → 기존 실서비스 상태로 되돌릴 때**

```bash
cd "C:\Users\yhs17\OneDrive\Desktop\마케팅사 홈페이지"
git fetch origin
git reset --hard origin/main
```

## 10. 정리

- **작업 폴더:** 항상 **‘마케팅사 홈페이지’** 한 폴더만 사용
- **수정·검증:**  
  저장 → `git add .` → `git commit -m "설명"` → **`git push bellodev main`** → GitHub/Netlify에서 확인
- **실서비스 반영:**  
  문제 없다고 판단되면 **`git push origin main`** 한 번만 실행

이 순서만 지키면, **bello(실서비스)는 건드리지 않은 채** bellodev에서만 먼저 수정·검증하고, 괜찮을 때만 실사이트에 반영할 수 있습니다.
